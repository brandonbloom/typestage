The painless way to generate TypeScript.

String templating Fragile

Direct, but you hand-manage separators, escaping, precedence, and hygiene. One stray comma and the output won't parse. 

__TEXT_TEMPLATE_CODE__
AST construction Verbose

Safe and well-formed, but a page of factory calls that looks nothing like the code it emits. 

__AST_CODE__
TypeStage Direct & safe

Looks like the output, composes like data, and stays hygienic — separators, arity, and binding handled for you. 

__TYPESTAGE_CODE__

Quotes let generated code look like code. Splices let generators compose that code with host TypeScript.

Syntax all the way down

Internally the generated program is real syntax: expressions are expressions, declarations are declarations, types are types.

❝❞

Syntax-aware quotes

Each quote form — q.expr, q.type, q.stmt, q.decl and more — chooses a TypeScript grammar entry point that constrains what can be spliced in.

⎘

Composable splices

${...} inserts a fragment as syntax. Arity is preserved, so generators hand expressions to each other without shipping separator or precedence rules.

✶

Automatic hygiene

TypeStage tracks bindings and free references. When an introduced local would capture a spliced reference, it renames the local for you.

⤓

Value persistence

Host-stage data — maps, sets, dates, regexps, plain objects — serializes into residual expressions for schemas, lookup tables, and config.

⌖

Source maps & origins

Attach origins with q.withOrigin so diagnostics and source maps can point back to an external source language.

⊞

Module graphs

Compile a local module graph from an entry file and emit one residual file per input module — organize compiler code normally.
